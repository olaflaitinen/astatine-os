# Code of Conduct

This project aligns with the spirit of Google Open Source community guidance and requires respectful, constructive, evidence-based collaboration.

## 1. Expected behavior

- Communicate professionally and focus on technical substance.
- Critique proposals, not individuals.
- Assume good intent and ask clarifying questions.
- Support inclusive participation across backgrounds and experience levels.

## 2. Unacceptable behavior

- harassment, intimidation, or discriminatory language
- repeated personal attacks
- doxxing or disclosure of private information
- malicious disruption of project collaboration spaces

## 3. Scope

Applies to:

- issue and pull request discussions
- repository comments and reviews
- project chat channels and events

## 4. Reporting channel

Report incidents to `conduct@astatine-os.org`.

Recommended report structure:

| Field | Description |
| --- | --- |
| Context | URL, channel, or event reference |
| Timestamp | date and time in UTC |
| Description | concise incident narrative |
| Evidence | logs, screenshots, links |

## 5. Enforcement model

| Severity | Typical action |
| --- | --- |
| Low | warning and behavior guidance |
| Medium | temporary interaction restrictions |
| High | removal from project spaces |

Enforcement is case-specific and aims to preserve community safety and constructive collaboration.

## 6. Confidentiality

Reports are handled with discretion and shared only with personnel required for investigation and remediation.

## 7. Acknowledgments

This policy is informed by:

- Google Open Source Community Guidelines
- open source conduct standards adapted to this repository
